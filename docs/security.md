# Обеспечение информационной безопасности (СЗИ)

Проект спроектирован с учетом требований регуляторов к защите КИИ (Критическая информационная инфраструктура) и промышленным системам (Industry 4.0).

## 1. Соответствие нормативной документации
При проектировании учитывались следующие нормативные акты:
*   **Приказ ФСТЭК №239:** Требования по обеспечению безопасности значимых объектов КИИ.
*   **Приказ ФСТЭК №235:** Порядок создания систем безопасности КИИ.
*   **ГОСТ Р 52447-2005:** Защита информации от несанкционированного доступа.

## 2. Модель угроз (Threat Model)

| Угроза | Метод нейтрализации |
| :--- | :--- |
| Подмена данных датчика (Spoofing) | Использование взаимной аутентификации (mTLS) на базе сертификатов. |
| Перехват трафика (Sniffing) | Шифрование полезной нагрузки MQTT через TLS 1.2/1.3. |
| DoS-атака на MQTT-брокер | Ограничение количества соединений с одного IP и использование IDS/IPS. |
| Несанкционированный доступ к БД | Разграничение прав доступа (RBAC) и аудит всех SQL-запросов. |

## 3. Сетевые средства защиты (СЗИ)
*   **Межсетевой экран (L7 Firewall):** Инспекция прикладного трафика (MQTT/HTTP).
*   **IDS/IPS:** Система обнаружения вторжений для анализа аномалий в сети IoT.
*   **VPN (ГОСТ):** Для удаленного доступа администраторов к системе используются шлюзы с поддержкой алгоритмов шифрования «Кузнечик»/«Магма» (требования ФСБ для защиты каналов).
