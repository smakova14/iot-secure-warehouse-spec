# Secure-IoT Warehouse: Система мониторинга Industry 4.0

Проект по проектированию защищенной системы интеллектуального мониторинга склада. Фокус проекта — на архитектуре сбора данных с IoT-устройств (MQTT), интеграции с WMS и обеспечении сетевой безопасности в соответствии с требованиями регуляторов (ФСТЭК/ФСБ).

## 🎯 Цель проекта
Разработать полный цикл проектной документации для MVP системы, которая контролирует состояние склада (датчики открытия дверей, температура, влажность) и защищает данные от подмены или перехвата в сегментированной сети.

## 🛠 Технологический стек (Проектируемый)
*   **Протоколы:** MQTT (TCP/IP), TLS 1.2/1.3, REST API.
*   **IoT-периферия:** ESP32, промышленные контроллеры.
*   **Backend:** Python (FastAPI) — разработка схем данных и логики обработки событий.
*   **Безопасность:** Средства криптографической защиты информации (СКЗИ), сегментация сети (VLAN), анализ соответствия приказам ФСТЭК №235, №239.
*   **Инструментарий аналитика:** Miro (C4 model), Draw.io (UML), Swagger (OpenAPI 3.0), Notion.

## 📂 Структура репозитория
*   `/docs` — Бизнес-требования (BRD) и Функциональные требования (SRS).
*   `/diagrams` — Архитектурные схемы: Sequence (взаимодействие компонентов), Activity (логика алармов), Deployment (сетевая топология).
*   `/api` — Спецификация взаимодействия между IoT-шлюзом и Backend-сервером (Swagger YAML).
*   `/security` — Модель угроз, план сегментации сети и маппинг требований ФСТЭК на компоненты системы.

## 🚀 Ключевые фичи (MVP)
1.  **Сбор телеметрии в реальном времени:** Обработка данных с датчиков по протоколу MQTT через защищенный брокер.
2.  **Детекция сетевых аномалий:** Логика выявления попыток несанкционированного подключения новых устройств к брокеру.
3.  **Интеграция с WMS:** Передача данных о критических инцидентах во внешнюю систему управления складом.
4.  **Compliance:** Проектирование системы с учетом защиты значимых объектов КИИ (Критическая информационная инфраструктура).
